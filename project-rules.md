# ข้อกำหนดระบบติดตามงานจัดซื้อจัดจ้าง (e-GP)

> **แหล่งอ้างอิงในโค้ด:** `src/lib/egp-milestones.ts` (`EGP_MILESTONES`, `EGP_MILESTONE_SHORT`, `EGP_STEP_LEGAL_HINTS`)

## แนวคิดระบบ (Concept)

**ProcureTrack ไม่ใช่ระบบคีย์ข้อมูลแทน e-GP** — แอปนี้ทำหน้าที่เป็น **Smart Dashboard และระบบแจ้งเตือนทุ่นแรง (Alerts)** สำหรับเจ้าหน้าที่พัสดุและผู้บริหาร

- ข้อมูลรายละเอียดทั้งหมดยังคีย์ใน **e-GP ของจริง** (e-bidding งานก่อสร้าง)
- แอปนี้เก็บเฉพาะสิ่งที่จำเป็นต่อการ **ติดตามความคืบหน้า** และ **ดักจับความผิดพลาด/ความเสี่ยง**

## หลักการเก็บข้อมูล (Data Scope)

โครงสร้าง **10 ขั้นตอนด้านล่าง** ใช้เป็น **Milestone หลัก** สำหรับ Timeline / Progress Bar บน Dashboard

**ต่อจากนี้ ช่องกรอกในแต่ละขั้นตอนจะเก็บเฉพาะ "คีย์เวิร์ดสำคัญที่ใช้คำนวณและดักจับความผิดพลาด"** — ไม่ต้องทำทุกช่องเหมือน e-GP

| ประเภทคีย์เวิร์ด | ตัวอย่างการใช้งาน |
|----------------|-------------------|
| **วันที่สำคัญ** | คุมเดดไลน์, แจ้งเตือนใกล้ครบกำหนด |
| **ตัวเลขงบ/ราคา** | เปรียบเทียบงบ vs ราคากลาง vs ราคาตกลง |
| **ชื่อบุคคล/คณะกรรมการ** | ตรวจไม่ให้ตั้งกรรมการซ้ำชุด |
